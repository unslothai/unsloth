# Apple Silicon resolver overrides.
#
# mlx-vlm 0.5.0 declares `transformers>=5.5.0` and mlx-lm 0.31.3 declares
# `transformers>=5.0.0` in their published metadata. Neither is true at
# their top-level import time -- mlx-vlm only loads AutoProcessor /
# AutoTokenizer / ProcessorMixin / BatchFeature which are stable across
# transformers 4.51+. The aggressive pin breaks the main venv's
# constraints.txt `transformers==4.57.6` and forces uv to backtrack
# unsloth (the user-reported downgrade).
#
# Override here relaxes both pins to transformers>=4.51.3, letting the
# main venv keep its 4.57.6 default. Per-model 5.x routing happens at
# runtime via the .venv_t5_530 / .venv_t5_550 side-cars
# (studio/backend/utils/transformers_version.py).
#
# Activated by exporting UV_OVERRIDE pointing at this file -- handled
# automatically by install.sh and studio/install_python_stack.py on
# darwin arm64 only.
transformers>=4.51.3
